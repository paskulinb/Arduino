1 /* DS1820 temperature sensor block
2  * The conversion process of DS1820 sensor takes at least 750ms by the specefifcations.
3  * This block divides the DS1820 sensor useage into two step:
4  * 1. Conversion issue by startConversion() method
5  * 2. Reading conversion result from sensor by readTemperatrue() method
6  * The result is stored in 'temperature' property of this class. The user
7  * can get this value by get() method.
8  *
9  * 1. Undependent usage of this block:
10  *
11  * startConversion();
12  * delay(800);
13  * if (dataReady() == DS1820_STAT_DATA_READY) {
14  * get(0, &temp);
15  * }
16  *
17  * Also possible:
18  *
19  * startConversion();
20  * while(dataReady() != DS1820_STAT_DATA_READY) {sleep(200);}
21  * get(0, &temp);
22  *
23  * 2. Block connection usage:
24  * The conversion is started through calling trigger() method.
25  * If trigger_caller_block pointer is passed to the method,
26  * the *trigger_caller_block will be called after conversion completed.
27  * In order to automatically read sensor and trigger the trigger_caller_block after conversion finished,
28  * this block has to be 'connected' to BlockTimer using setTimer() method.
29  *
30  * BlockTiming TIMING; //in global scope
31  *
32  * //In your SETUP procedure:
33  * BlockBase* DS1820 = new BSensorDS1820(12, TIMING); //DS1820 will create his own Timer using BlockTiming functionality
34  *
35  * //In the infinite LOOP must be:
36  * TIMING.update();
37  */
